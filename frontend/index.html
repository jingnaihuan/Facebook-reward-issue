最关键：用 http://localhost:18765 打开本页，不要用 127.0.0.1——FB 的应用域名不接受 IP，用 IP 一定登录失败。双击「启动.command」现已自动用 localhost。

① 创建自己的 Facebook App（已有就跳过这步）

  • 到 developers.facebook.com 用你的 Facebook 账号登录（首次会引导注册成开发者）→ 右上「我的应用」→ 创建应用
  • 应用名称随意（如 RewardHub），联系邮箱填自己的
  • 用例 (use case) 必须至少选一个，认准 「管理公共主页的所有内容 / Manage everything on your Page」——不能随便选：用例决定后台能添加哪些权限，只有这个用例能加上本工具要的三个粉专权限
  • 若中途提示关联商务资产组合 (business portfolio)，能跳过就跳过，用不到
  • 创建完成后最关键一步：进入该用例的 「定制 / Customize」→ 权限 (Permissions)，把 pages_show_list pages_read_engagement pages_read_user_content 这三个一次性全部添加 / 启用，缺一个都不行——别等登录报 Invalid Scopes 错误再回来补。
  • 回到应用面板，顶部的 App ID 就是左边输入框要填的那串数字

接着在 developers.facebook.com → 你的 App 后台做以下设置（一次性）：

② 设置 → 基本 (Settings · Basic)

  • 底部「+ 添加平台」→ 选「网站」→ 站点网址 http://localhost:18765/
  • 应用域名 (App Domains) 不用填——它要求带 .com/.org 之类后缀、不接受纯 localhost，留空也能正常抓取。

③ 产品加「Facebook 登录」→ 设置

  • 使用 JavaScript SDK 登录 = 是
  • JavaScript SDK 允许域名：http://localhost:18765
  • 客户端 / 网页 OAuth 登录 = 是
  • 有效 OAuth 重定向 URI 不用填——开发模式下 http://localhost 的跳转会自动放行（后台也会这么提示），留空即可。
  • 左侧产品列表里找不到「Facebook 登录」的设置入口？到用例的「定制 / Customize」里找「设置 (Settings)」，是同一组配置

④ 应用模式 / 角色

  • 保持开发模式即可（你是 App 管理员，直接能用）。别尝试把 App 上线：上线要对 pages_read_user_content 等权限做 App Review + 商家验证，内部小工具过不了审，也完全没必要——开发模式就是长期形态。
  • 让同事也用，两个条件缺一不可：⑴ 应用面板「应用角色」里把同事的 FB 账号加为「测试人员 / 开发者」（对方需接受邀请）；⑵ 同事本人得是目标粉专的管理员 / 编辑——不然授权成功也读不到粉专数据。

⑤ 权限

在 ① 的用例定制里添加好的 pages_show_list pages_read_engagement pages_read_user_content 三个权限，登录时本工具会自动向你请求授权，点同意即可。开发模式 + App 角色成员 + 只读自己管理的粉专 → 属标准访问，无需 App Review。

使用流程

  1. 填 App ID → 点「初始化」
  2. 点「用 Facebook 登录」→ 授权（弹窗被浏览器拦的话点允许）
  3. 选粉专 → 载入贴文 / 填贴文 ID → 抓取

Token 过期了怎么办

方式 A 的 Token 也是短效（约 1–2 小时），但过期后再点一次「用 Facebook 登录」即可——浏览器留着 FB 会话，通常一键续期、免重新授权。这就是它比方式 B 省事的地方。

到 Graph API Explorer 勾选 pages_show_list、pages_read_engagement、pages_read_user_content，生成后切成「Page Access Token」选你的粉专再复制。建议换成长效 Token（约 60 天）省得频繁重新生成。
