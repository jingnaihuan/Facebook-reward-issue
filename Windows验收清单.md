# Windows 真机验收清单

> 开发机为 Mac，以下项目**无法在 Mac 上真实完成**（`.bat` 启动、Python 路径探测、原生对话框弹出等属操作系统层面），需在一台 **Windows 10/11** 机器上人工跑一遍。逐项打勾即可。
>
> 前置：装好 **Python 3.9+**，安装时勾选「Add python.exe to PATH」。

## A. 启动

- [ ] 把整个 `社群互动发奖中台` 文件夹拷到 Windows 机器。
- [ ] 双击 `启动.bat`，弹出命令行窗口，显示中文「正在启动 社群互动发奖中台…」**不乱码**。
- [ ] 首次运行自动安装 `openpyxl`（若已装则跳过），无红色报错。
- [ ] 数秒后**自动打开默认浏览器**，地址为 `http://localhost:8765`，页面正常显示。
- [ ] 右上角状态显示「服务已就绪」（绿点）。
- [ ] 关闭命令行窗口后，浏览器刷新页面应连不上（服务已停）——确认关窗即停服。

### 若未装 Python
- [ ] 在没装 Python 的机器上双击 `启动.bat`，应显示中文错误提示并给出下载网址，窗口停住（`pause`）不闪退。

## B. 五步流程（用「粘贴文本」免外部依赖跑通）

- [ ] 步骤1：点「粘贴文本」，粘入几行 `序号<Tab>内容(含10位玩家ID)<Tab>按赞<Tab>时间`，点「解析文本」，「下一步」变亮。
- [ ] 步骤2：进入后「留言总数 / 命中 ID / 无有效 ID / 去重后玩家」数字正确；切换去重策略数字随之变化。
- [ ] 步骤3（可选，需 Eastblue 登录）：点「按玩家ID拉取玩家表」，首次会弹浏览器要求登录 Eastblue；进度文字**中文不乱码**（这是本次重点修复项）。无凭证可跳过此步。
- [ ] 步骤4：勾选目标语言，人数统计正确。
- [ ] 步骤5：叠加一个奖项（如「前 N 楼」），点导出。

## C. 导出（重点：Windows 原生保存框）

- [ ] 点导出后，弹出 Windows 原生「另存为」对话框（tkinter），默认文件名形如 `发奖名单-YYYYMMDD.xlsx`。
- [ ] 选路径保存 → 提示成功，用 Excel 打开该 xlsx，各 sheet（特殊奖 / 参与奖 / 无效）内容正确、中文表头正常。
- [ ] 再点一次导出，在对话框点「取消」→ 页面提示「已取消导出」，不报错、不生成文件。

## D. 配置预设

- [ ] 在步骤3/4 配好后，输入预设名 → 「保存当前配置」→ 刷新页面 → 预设仍在，可载入。
- [ ] 预设文件位于 `C:\Users\<你>\.reward_hub_app\presets.json`。

## 发现问题时的排查提示

- 中文乱码：确认 `启动.bat` 顶部 `chcp 65001` 生效；命令行字体建议用「新宋体/Consolas」。
- `py` / `python` 找不到：确认安装时勾了 Add to PATH，或在命令行 `where py` / `where python` 验证。
- 保存框弹不出来：极少数精简版 Python 未装 tkinter，此时会自动落到 `我的文档\发奖中台工作区`，属预期回退。
