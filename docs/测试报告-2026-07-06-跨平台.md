## 二、改造内容（对应真实缺陷）

1. **子进程 UTF-8 编码 Bug**（真实缺陷）：`server.py` 读 Eastblue 子进程输出未指定编码，中文版 Windows（cp936）会乱码 / JSON 解析失败。已改为 `encoding="utf-8", errors="replace"`。
2. **保存对话框跨平台**：抽出 `reward_hub/platform_util.py`，Mac=osascript，Windows=tkinter，其他/失败=回退工作区。
3. **启动脚本**：新增 `启动.bat`（`chcp 65001` + `py`/`python` 探测 + 依赖安装 + `PYTHONUTF8=1`）；`启动.command` 同步简化。
4. **开浏览器统一**：由 `server.py` 用 `webbrowser` 打开 `localhost`，两端一致，脚本不再各自 `open`/`start`。

核心业务模块（extract_id / dedup / language_filter / rule_engine / export / eastblue_parse / config_store）**未改业务逻辑**。
